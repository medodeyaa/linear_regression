# Ford Car Price Prediction[cite: 1]

## Overview
This project builds a Machine Learning model using Linear Regression to predict the price of Ford cars[cite: 1]. The analysis and modeling are conducted using Python, with data manipulation, visualization, and machine learning libraries[cite: 1]. 

The dataset used for this project is sourced from Kaggle: [Ford Car Price Prediction Dataset](https://www.kaggle.com/datasets/adhurimquku/ford-car-price-prediction?resource=download)[cite: 1].

---

## Dataset Details
The initial dataset contains **17,966 records** and **9 columns**[cite: 1]. 
* **Features Used**: `model`, `year`, `transmission`, `mileage`, `fuelType`, `tax`, `mpg`, and `engineSize`[cite: 1].
* **Target Variable**: `price`[cite: 1].

---

## Project Workflow

### 1. Data Cleaning[cite: 1]
To ensure the model is trained on high-quality data, several preprocessing steps were taken:
* Checked for missing values (none were found)[cite: 1].
* Identified and dropped **154 duplicate rows**[cite: 1].
* Addressed zero-values in the `tax` and `engineSize` columns by replacing them with the column means[cite: 1].
* Removed extreme outliers in the `price` column using the Interquartile Range (IQR) method, resulting in a cleaner dataset of **17,530 rows**[cite: 1].

### 2. Exploratory Data Analysis (EDA)[cite: 1]
Visualizations were created to understand the underlying patterns in the data:
* **Correlation Matrix**: A heatmap to show relationships between numerical variables[cite: 1].
* **Price Distribution**: A histogram showing the spread of car prices[cite: 1].
* **Average Price by Fuel Type**: A bar plot comparing the average price of cars based on their fuel type[cite: 1].

### 3. Data Preprocessing & Modeling[cite: 1]
* **Feature Encoding**: Categorical features were converted into numerical values using one-hot encoding (`pd.get_dummies` with `drop_first=True`)[cite: 1].
* **Data Splitting**: The data was split into an 80% training set and a 20% testing set[cite: 1].
* **Algorithm**: A standard `LinearRegression` model from `scikit-learn` was trained on the data[cite: 1].
* **Post-processing**: Any negative price predictions generated by the model were clipped to 0 to make logical sense in a real-world context[cite: 1].

---

## Model Evaluation[cite: 1]
The model performed strongly, explaining approximately 85% of the variance in Ford car prices[cite: 1]. The exact evaluation metrics on the test set are as follows:

| Metric | Value |
| :--- | :--- |
| **Mean Absolute Error (MAE)** | 1274.03[cite: 1] |
| **Mean Squared Error (MSE)** | 2615711.95[cite: 1] |
| **Root Mean Squared Error (RMSE)** | 1617.31[cite: 1] |
| **R-squared ($R^2$)** | 0.849[cite: 1] |

A scatter plot comparing actual vs. predicted prices was also plotted to visually confirm the model's accuracy[cite: 1].
